- POST handler accepting JSON: { productDescription: string, competitor?: string }
- Validate with Zod: productDescription required (min 5, max 500 chars), competitor optional (max 100 chars)
- Rate limiting: in-memory Map keyed by IP (from headers x-forwarded-for or request.ip)
  - Track { count, resetAt } per IP
  - Max 3 requests per hour
  - Return 429 with { error: "Try again in a few minutes" } when exceeded
- Build search queries from product description:
  - Split description into 2-3 keyword phrases
  - If competitor provided, add "alternative to {competitor}" query
- Use snoowrap adapter to search Reddit (if available) OR create a simplified Reddit search:
  - Use Reddit's public JSON API: fetch(`https://www.reddit.com/search.json?q=${encodeURIComponent(query)}&sort=new&limit=10&t=week`)
  - Parse results into simplified signal format
  - No auth needed for public search
- Classify results using structural matching (import from monitoring lib if compatible) or simple keyword matching
- Return { signals: SimplifiedSignal[], count: number }
- Max 10 results
- Structured logging with correlationId
- Timeout: abort after 8 seconds, return partial results
