1. Install Stripe: `pnpm add stripe @stripe/stripe-js`

2. Create `src/features/billing/actions/checkout.ts` ("use server"):
- `createCheckoutSession(priceId: string, mode: "subscription" | "payment")`:
  - Get authenticated user
  - Get or create Stripe customer (query users.stripe_customer_id, if null create via stripe.customers.create with email + metadata.supabase_user_id, save to users table)
  - Create Stripe Checkout Session:
    - customer: customerId
    - line_items: [{ price: priceId, quantity: 1 }]
    - mode: mode parameter
    - success_url: `${process.env.NEXT_PUBLIC_SITE_URL}/billing?success=true`
    - cancel_url: `${process.env.NEXT_PUBLIC_SITE_URL}/billing?canceled=true`
    - For credit pack payments: add metadata.credit_pack_credits with the pack credits amount (lookup from CREDIT_PACKS by priceId)
  - redirect(session.url!)

- `startFreeTrial()`:
  - Get authenticated user
  - Check if trial_ends_at is already set (prevent duplicate trials)
  - Update users: trial_ends_at = now() + 3 days, credits_balance = 500
  - Insert credit_transaction: type = 'monthly_grant', amount = 500, description = 'Free trial credits'
  - redirect("/")

3. Create `src/features/billing/actions/manage-subscription.ts` ("use server"):
- `cancelSubscription()`:
  - Get authenticated user, get stripe_customer_id
  - List active subscriptions via stripe.subscriptions.list({ customer })
  - Cancel at period end: stripe.subscriptions.update(subId, { cancel_at_period_end: true })
  - Return { success: true, endsAt: subscription.current_period_end }

- `getInvoices()`:
  - Get authenticated user, get stripe_customer_id
  - If no customer ID, return []
  - stripe.invoices.list({ customer, limit: 20 })
  - Return array of { id, amount_paid, currency, status, created, invoice_pdf, period_start, period_end }

4. Create `src/app/api/stripe/webhook/route.ts`:
- CRITICAL: Use `await request.text()` for body (NOT request.json())
- Verify signature with stripe.webhooks.constructEvent(body, signature, STRIPE_WEBHOOK_SECRET)
- Create service role Supabase client inline
- Handle events:
  - `checkout.session.completed`:
    - Get customer ID and supabase_user_id from session metadata or customer lookup
    - If mode === "subscription": update users SET subscription_active = true, billing_period = (lookup from price metadata)
    - If mode === "payment" (credit pack): lookup credit_pack_credits from session metadata, call add_credits RPC
  - `customer.subscription.updated`:
    - Update users billing_period and subscription_active based on subscription status
    - If cancel_at_period_end === true, log but keep active until period end
  - `customer.subscription.deleted`:
    - Update users SET subscription_active = false, billing_period = null
  - `invoice.payment_failed`:
    - Log to job_logs with metadata including invoice ID and customer email
    - Could trigger email notification (future)
- Structured logging with correlationId throughout
- Return NextResponse.json({ received: true })
