## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, card internal gaps |

## Interaction Contracts

### Onboarding Flow

| Interaction | Behavior |
|-------------|----------|
| Question navigation | Forward only with "Next" button, no back navigation. Enter key submits. |
| Input validation | Step 1 and 2 require non-empty input (min 5 chars). Step 3 is optional. |
| Transition between steps | Fade out current, fade in next. 200ms duration. |
| Scan animation | After step 3, auto-start. Show typing animation for 3-5 seconds. |
| Results reveal | Fade in all signal cards simultaneously after animation completes. |
| Post-onboarding redirect | Navigate to `/` (dashboard) with `?onboarded=true` query param to show checklist. |

### Kanban Pipeline

| Interaction | Behavior |
|-------------|----------|
| Drag-and-drop | Cards draggable between columns. Visual placeholder shown in target column. |
| Mobile fallback | "Move to..." dropdown on each card (no drag on touch screens below 768px). |
| Column scroll | Each column scrolls vertically independently. Board scrolls horizontally on mobile. |
| Card click | Navigate to `/prospects/[id]` full page detail. |
| Stage update | Optimistic UI update on drop. Revert on server error with toast. |

### Credit Balance Widget (Sidebar)

| Interaction | Behavior |
|-------------|----------|
| Default state | White/foreground text, normal weight. |
| Warning (<100) | Text turns warning orange. |
| Critical (<50) | Text turns destructive red. |
| Click | Navigate to `/billing`. |
| Hover | Underline, pointer cursor. |

### Billing Page

| Interaction | Behavior |
|-------------|----------|
| Plan selection | Click plan card to redirect to Stripe Checkout. Current plan shown with "Current plan" badge. |
| Period toggle | Toggle between monthly/quarterly/annual. Savings badge updates. |
| Credit pack purchase | Click "Buy" on pack card to redirect to Stripe Checkout. |
| Cancel subscription | Click "Cancel" opens AlertDialog confirmation. Confirm redirects to server action. |
| Invoice list | Paginated table, 10 per page. Click row opens invoice PDF in new tab. |

### /live Page

| Interaction | Behavior |
|-------------|----------|
| Polling | Fetch new signals every 10 seconds. Prepend new cards with fade-in. No full page refresh. |
| Scroll | Infinite scroll, load 20 signals at a time. |
| CTA | "Sign up free" links to `/login` (existing auth page). |

### "Scan My Product" Hook

| Interaction | Behavior |
|-------------|----------|
| Submit | Disable button, show "Scanning Reddit..." loading text. Timeout after 10 seconds. |
| Results | Fade in signal cards below the form. Show count heading. |
| CTA after results | "Sign up to contact them" links to `/login`. |
| Rate limiting | 3 scans per IP per hour. Show "Try again in a few minutes" on limit hit. |

### Weekly Results Card

| Interaction | Behavior |
|-------------|----------|
| Download | Generate PNG via canvas/SVG, trigger browser download. |
| Share to X | Open new window with pre-filled tweet text + image URL. |
| Share to LinkedIn | Open new window with pre-filled LinkedIn post + image URL. |

## Layout Contracts

### Onboarding Screens (`/onboarding`)

- Full-screen centered layout (no sidebar, no header)
- Max-width 480px for question content
- Vertical center alignment
- Step indicator dots at top (3 dots, filled = complete/current, outlined = upcoming)
- Primary accent color for current step dot

### /billing Page

- Standard app shell (sidebar + header)
- Two-column layout above 1024px: left column for plan/packs (2/3 width), right column for balance summary (1/3 width)
- Single column stack below 1024px
- Tabs for navigation: Plans, Credit Packs, History

### /prospects Page (Kanban)

- Standard app shell (sidebar + header)
- Full-width kanban board below page header
- 6 columns, each 280px min-width
- Horizontal scroll on viewport < 1680px
- Page header: "Prospects" heading + "Export CSV" button right-aligned
- Column header: stage name + count badge

### /prospects/[id] Page (Detail)

- Standard app shell (sidebar + header)
- Back button "Back to prospects" at top
- Two-column layout above 1024px: left (2/3) for conversation history, right (1/3) for prospect info + notes + tags
- Single column stack below 1024px

### /live Page

- No app shell (public page, no sidebar/header)
- Custom minimal header: repco logo left, "Sign up free" CTA right
- Hero section: centered heading + subheading + stat bar
- Signal feed: centered, max-width 640px, vertical scroll
- "Scan my product" section: below stats or as a separate section, centered, max-width 480px

### Dashboard Additions

- Onboarding checklist card: insert as first card in dashboard grid when `?onboarded=true` or checklist incomplete
- Credit summary card: add to existing dashboard card grid
- Prospect stats card: add to existing dashboard card grid (total, replied, converted, est. revenue)

### Sidebar Updates

- Add "Prospects" nav item with Users icon (already exists in NAV_ITEMS, activate href to `/prospects`)
- Add "Billing" nav item with CreditCard icon, href `/billing`
- Credit balance widget in SidebarFooter, above user email
