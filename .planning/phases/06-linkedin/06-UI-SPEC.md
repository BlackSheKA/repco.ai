## Signal Card -- LinkedIn Variant

The signal card conditionally renders LinkedIn-specific content when `signal.platform === "linkedin"`.

### Layout Specification

```
+----------------------------------------------------------+
| [LinkedIn]  Author Full Name  ·  Professional Headline   |  <- top row
|                                                   2h ago |
|                                                          |
| Post excerpt text, up to 3 lines with line-clamp-3...    |  <- middle row
|                                                          |
| [flame 7/10]    View on LinkedIn ^   [Connect] [Dismiss] |  <- bottom row
+----------------------------------------------------------+
```

### Top Row Details

- **Badge**: `bg-[#0A66C2] text-white` rounded-full, height 24px (h-6), text-sm font-medium -- matches Reddit badge dimensions exactly
- **Author name**: `text-sm font-medium text-muted-foreground` -- shows full name (not handle)
- **Professional headline**: `text-sm text-muted-foreground` after a middot separator -- truncated with `truncate` class, max-width 60% of row
- **Time ago**: right-aligned, `text-sm text-muted-foreground`, same as Reddit variant

### Bottom Row Details

- **CTA label**: "Connect" instead of "Contact" for LinkedIn signals (reflects the two-step outreach: connect first, DM after acceptance)
- **View link**: "View on LinkedIn" with ExternalLink icon, same styling as "View on Reddit"

### Platform Detection Logic

```
if signal.platform === "linkedin":
  badge color = #0A66C2, label = "LinkedIn"
  author display = signal.author_handle (full name from Apify)
  headline display = signal.author_headline (professional headline)
  CTA label = "Connect"
  view link text = "View on LinkedIn"
else:
  badge color = #FF4500, label = "Reddit"
  author display = "u/" + signal.author_handle
  headline display = none
  CTA label = "Contact"
  view link text = "View on Reddit"
```

## Staleness Warning Banner

Appears above the signal feed when the last successful LinkedIn monitoring run exceeds 8 hours (2x the 4-hour cadence).

### Layout Specification

```
+----------------------------------------------------------+
| [!] LinkedIn check failed -- last successful: 6h ago     |
+----------------------------------------------------------+
```

### Visual Specification

| Property | Value |
|----------|-------|
| Background | `bg-amber-50` (light) / `bg-amber-950/30` (dark) |
| Border | `border border-amber-200` (light) / `border-amber-800/50` (dark) |
| Text | `text-amber-800` (light) / `text-amber-200` (dark) |
| Icon | Phosphor `Warning` icon, 16px, `text-amber-600` (light) / `text-amber-400` (dark) |
| Padding | 12px vertical, 16px horizontal |
| Border radius | `rounded-lg` (matches card radius) |
| Position | Above filter bar, below terminal header, full width of feed column |
| Dismiss | No dismiss button -- banner auto-clears when next successful run completes |

### Copy

| State | Copy |
|-------|------|
| Stale (8-12h) | "LinkedIn monitoring delayed -- last successful check: {X}h ago" |
| Failed (12h+) | "LinkedIn monitoring failed -- last successful check: {X}h ago. Retrying automatically." |

## Interaction States

|-------|--------|

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
