# Phase 17 — UI Design Contract

> Visual and interaction contract for Phase 17: Residential Proxy + GoLogin Profile Allocator.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 17 is backend-heavy. The entire allocator, GoLogin REST calls, and country mapping are
invisible to the user. The **only UI surface** is the existing connect-account flow on `/accounts`,
which must cover the ~3–8 second allocation window that now precedes the GoLogin browser launch.

No new screens. No proxy/profile/fingerprint terminology exposed. The user clicks a button,
sees a short "Setting up your account..." state, then arrives at the existing login step exactly
as before.

## State Map: Connect Flow

The connect flow has three sequential UI states. Phase 17 extends state A by lengthening its
duration (~0.5s today → ~3–8s with allocation). States B and C are unchanged.

### State A — Allocation pending (NEW extended duration)

**Trigger:** User clicks "Connect Reddit Account" or "Connect LinkedIn Account"

**What renders:**

Reddit path — existing handle-form card is shown first. On submit:
```
[Card]
  [CardContent]
    [Loader2 animate-spin h-5 w-5 text-muted-foreground]
    <p class="text-base">Setting up your account...</p>
```

LinkedIn path — immediately on button click (no handle form):
```
[Card]
  [CardContent flex items-center gap-3 py-6]
    [Loader2 animate-spin h-5 w-5 text-muted-foreground]
    <p class="text-base">Setting up your account...</p>
```

**Submit button state (Reddit handle form during submission):**
```
<Button type="submit" disabled>
  <Loader2 class="mr-2 h-4 w-4 animate-spin" />
  Setting up your account...
</Button>
```

**Duration:** ~3–8 seconds (GoLogin proxy alloc + profile create + fingerprint patch)

**Rules:**
- No percentage progress indicator — duration is non-deterministic
- No "proxy", "profile", "fingerprint", "GeoProxy", "browser profile" copy visible to user
- Spinner is `Loader2` from existing lucide-react import — no new icon dependency

### State B — Browser login (unchanged)

**Trigger:** Allocation completes; `connectAccount` returns `{ accountId, profileId }`

**What renders:** Existing `ConnectionFlow` component, step 1 — "Open remote browser" + login URL
copy block. No changes in Phase 17.

### State C — Verified / Error (unchanged)

**Trigger:** `verifyAccountSession` completes

**What renders:** Existing `ConnectionFlow` step 3 — `CheckCircle` (success) or `AlertTriangle`
(failure). No changes in Phase 17.

## Error State Detail

### Allocation failure (new in Phase 17)

**Rendered in:** `account-list.tsx`, same inline position as the allocation pending card.

**Trigger:** `connectAccount` server action returns `{ error: "Could not set up the account right now — please try again in a moment." }`

**Rendering:** Existing `toast.error(result.error)` call — Sonner toast in the bottom-right.
No inline card error state required; the pending card dismisses and the toast appears.
This matches the existing pattern already in `openConnectDialog` for LinkedIn.

**Recovery:** User can click "Connect Reddit Account" / "Connect LinkedIn Account" again.
No auto-retry. No countdown.

## Interaction Contracts

### Double-submit guard (D-09 mitigation)

The submit button and "Connect X Account" buttons must be `disabled` immediately on first click
and remain disabled until the server action resolves. This is the only race-protection layer
per CONTEXT D-09.

Implementation: existing `setSubmitting(true)` pattern in `account-list.tsx` already does this
for the Reddit handle form submit. LinkedIn path already disables via `submitting` state.
No new logic required — only copy changes.

### Spinner

`Loader2` from `lucide-react` with `animate-spin` class. Already imported in `account-list.tsx`.
Size: `h-5 w-5` in the pending card (matches existing LinkedIn pending card). `h-4 w-4` inside
the disabled submit button (matches existing "Creating profile..." button).

### Touch targets

All interactive elements (buttons) meet 44px minimum touch target height via shadcn Button
defaults. No exceptions in this phase.
