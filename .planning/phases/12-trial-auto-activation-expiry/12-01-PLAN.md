SECURITY DEFINER SET search_path = ''

  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

**Part 2 — One-time backfill UPDATE:**

```sql
UPDATE public.users
SET
  trial_ends_at = NOW() + INTERVAL '3 days',
  credits_balance = GREATEST(credits_balance, 500),
  updated_at = NOW()
WHERE trial_ends_at IS NULL
  AND subscription_active = false;
```

**Part 3 — Backfill credit_transactions INSERT:**
Insert a matching credit_transactions row for each user touched by Part 2. Use an INSERT ... SELECT from users where the grant was just applied. Because the UPDATE and INSERT are in the same migration, use a condition that identifies the same set:

```sql
INSERT INTO public.credit_transactions (user_id, type, amount, description, created_at)
SELECT id, 'monthly_grant', 500, 'Trial backfill', NOW()
FROM public.users
WHERE trial_ends_at IS NOT NULL
  AND subscription_active = false
  AND updated_at >= NOW() - INTERVAL '5 seconds'
ON CONFLICT DO NOTHING;
```

Note: The `ON CONFLICT DO NOTHING` prevents duplicate rows on idempotent re-runs. The `updated_at >= NOW() - 5 seconds` window is narrow enough to target only the backfill batch.
