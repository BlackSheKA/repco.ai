# Phase 19 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Scope Note

Phase 19 is a **backend/schema-only** phase. Work consists of:

1. New ENUMs `subscription_plan`, `billing_cycle` (DB migration)
2. New `users` columns: `subscription_plan`, `billing_cycle`, `credits_included_monthly`, `credits_balance_cap` (DB migration)
3. `handle_new_user` trigger rewrite (DB migration)
4. New `signup_audit` table + RLS (DB migration)
5. `PLAN_CONFIG` TS const at `src/features/billing/lib/plan-config.ts`
6. Signup server action passes `{ ip }` via `supabase.auth.signUp({ options: { data: { ip } } })` (server-only change, no visible UI delta)
7. `/auth/callback/route.ts` captures IP and upserts `signup_audit` for OAuth signups (server-only)

**No new user-facing screens, components, copy, or interactions are introduced.** The existing `/login` page (Phase 1) and `/onboarding` wizard (Phase 5) continue to function unchanged. New users land on the free plan invisibly — no "welcome to free tier" UI, no credit-grant toast, no plan badge changes (Phase 21 owns billing UI refactor).

The contract below is therefore a **passthrough**: it documents that this phase inherits the existing design system unchanged and adds no new visual surface area. Checker should validate "no new UI" rather than re-validating tokens.
