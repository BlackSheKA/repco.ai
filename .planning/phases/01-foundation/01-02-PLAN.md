1. Create `supabase/migrations/00003_rls_policies.sql`:

   Enable RLS on ALL 11 tables. Then create policies following these rules:

   **users:**
   - SELECT: `auth.uid() = id` (users read own row)
   - UPDATE: `auth.uid() = id` (users update own row)
   - No INSERT policy (trigger handles creation)
   - No DELETE policy (users cannot self-delete)

   **credit_transactions:**
   - SELECT: `auth.uid() = user_id` (read own transactions)
   - No INSERT/UPDATE/DELETE for authenticated users (server-side only via service_role)

   **monitoring_signals:**
   - SELECT: `auth.uid() = user_id`
   - INSERT: `auth.uid() = user_id`
   - UPDATE: `auth.uid() = user_id`
   - DELETE: `auth.uid() = user_id`

   **product_profiles:**
   - SELECT: `auth.uid() = user_id`
   - INSERT: `auth.uid() = user_id`
   - UPDATE: `auth.uid() = user_id`
   - No DELETE (protect profile data)

   **social_accounts:**
   - SELECT: `auth.uid() = user_id`
   - INSERT: `auth.uid() = user_id`
   - UPDATE: `auth.uid() = user_id`
   - No DELETE (lifecycle managed server-side)

   **intent_signals:**
   - SELECT: `auth.uid() = user_id` for authenticated; also allow anon SELECT WHERE `is_public = true` (for /live page)
   - INSERT: `auth.uid() = user_id` (server inserts via service_role, but policy allows it)
   - UPDATE: `auth.uid() = user_id` (for dismiss/action status changes)

   **prospects:**
   - SELECT: `auth.uid() = user_id`
   - INSERT: `auth.uid() = user_id`
   - UPDATE: `auth.uid() = user_id`
   - No DELETE

   **actions:**
   - SELECT: `auth.uid() = user_id`
   - INSERT: `auth.uid() = user_id`
   - UPDATE: `auth.uid() = user_id` (for approval/rejection)
   - No DELETE

   **action_counts:**
   - SELECT: `auth.uid() = (SELECT user_id FROM social_accounts WHERE id = account_id)` (read via account ownership)
   - No INSERT/UPDATE/DELETE for authenticated users (server-side via service_role)

   **live_stats:**
   - SELECT: allow all (public data for /live page, no auth.uid() check)
   - No INSERT/UPDATE/DELETE for authenticated users (server-side only)

   **job_logs:**
   - SELECT: `auth.uid() = user_id` (users can read their own job logs)
   - No INSERT/UPDATE/DELETE for authenticated users (server-side via service_role)

2. Create `supabase/migrations/00004_auth_trigger.sql`:

   ```sql
   -- Sync auth.users -> public.users on signup
   CREATE OR REPLACE FUNCTION public.handle_new_user()
   RETURNS trigger
   LANGUAGE plpgsql
   SECURITY DEFINER SET search_path = ''
   AS $$
   BEGIN
     INSERT INTO public.users (id, email, created_at, updated_at)
     VALUES (
       NEW.id,
       NEW.email,
       NOW(),
       NOW()
     );
     RETURN NEW;
   END;
   $$;

   CREATE OR REPLACE TRIGGER on_auth_user_created
     AFTER INSERT ON auth.users
     FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
   ```

   CRITICAL: The function uses `SECURITY DEFINER` to bypass RLS (since there's no INSERT policy on users). The `SET search_path = ''` prevents search_path injection attacks.
