## Typography

| Role | Font | Size | Weight | Line Height |
|------|------|------|--------|-------------|
| Body | Inter | 16px | 400 (regular) | 1.5 |
| Label | Inter | 14px | 500 (medium) | 1.4 |
| Heading | Instrument Serif | 28px | 400 (regular) | 1.2 |
| Display | Instrument Serif | 40px | 400 (regular) | 1.1 |
| Mono | JetBrains Mono | 14px | 400 (regular) | 1.6 |

Notes:
- Instrument Serif is a display typeface -- only use it for headings and the brand name. Never for body text or labels.
- JetBrains Mono is reserved for the terminal header (Phase 2+), code snippets, and technical identifiers.
- Inter at weight 500 is used for labels, nav items, and button text. Weight 600 (semibold) is used only for emphasis within body text.

## Screens

### Screen 1: Login Page (`/login`)

**Layout:** Split horizontal -- 50/50 on desktop (min-width 1024px), full-width stacked on mobile.

**Left panel (brand):**
- Background: `#09090B` (always dark, regardless of theme)
- Brand name "repco" in Instrument Serif at 40px, color `#FFFFFF`
- Tagline: "Your AI sales rep that never sleeps." in Inter 16px, color `#A1A1AA`
- Vertically and horizontally centered
- No images, no illustrations -- text only

**Right panel (auth form):**
- Background: dominant surface color (theme-aware)
- Vertically and horizontally centered, max-width 400px
- Heading: "Welcome back" in Instrument Serif 28px
- Subheading: "Sign in to your account" in Inter 16px, muted text color
- 24px gap between heading group and form fields

**Form elements (top to bottom, 16px gaps):**
1. Email input (shadcn `Input`, label "Email address", type email)
2. "Continue with email" button (shadcn `Button`, variant default, full width, accent background)
3. Divider with centered text "or" (horizontal rule with text overlay, muted color)
4. "Continue with Google" button (shadcn `Button`, variant outline, full width, Google icon left-aligned)

**States:**
- Loading: "Continue with email" button shows spinner icon, disabled state
- Error: red border on email input, error message below in 14px destructive color
- Success (magic link sent): replace form with confirmation message: "Check your email" heading + "We sent a magic link to {email}. Click it to sign in." body

### Screen 2: App Shell (authenticated, `/`)

**Layout:** Sidebar + header + main content area.

**Sidebar:**
- Width: 240px on desktop, collapsed to 0 on mobile (hamburger toggle in header)
- Background: secondary surface color
- Top: brand mark "repco" in Instrument Serif 20px, 24px padding from top and left
- Nav items: Inter 14px weight 500, 8px vertical padding, 16px horizontal padding, rounded-md
- Active nav item: accent color left border (3px), text inherits accent color
- Nav items for Phase 1 (placeholder, non-functional): Dashboard, Signals, Approvals, Prospects, Accounts, Settings
- Bottom: user email truncated in Inter 14px muted, "Sign out" text button in 14px

**Header:**
- Height: 48px
- Background: dominant surface color
- Left: hamburger menu icon (mobile only, hidden on desktop)
- Right: theme toggle (sun/moon icon) + user avatar placeholder (circle, 32px, initials)
- Bottom border: 1px border color

**Main content area:**
- Background: dominant surface color
- Centered placeholder content, max-width 600px:
  - Heading: "Welcome to repco" in Instrument Serif 28px
  - Body: "Your workspace is being set up. Features will appear here as they're built." in Inter 16px, muted text
  - No CTA button -- this is a placeholder, not an empty state with an action

## Interaction Contracts

### Login flow
1. User lands on `/login` -- sees split layout
2. User enters email, clicks "Continue with email" -- button enters loading state (spinner, disabled)
3. On success: form replaced with "Check your email" confirmation (no page navigation)
4. On error: toast notification with error message, button re-enabled
5. User clicks magic link in email -- redirected to `/` (app shell)
6. If already authenticated and visits `/login` -- redirect to `/`

### Google OAuth flow
1. User clicks "Continue with Google" -- redirect to Google consent screen
2. On success: redirect to `/` (app shell)
3. On error: redirect back to `/login` with error toast

### Sign out
1. User clicks "Sign out" in sidebar bottom
2. Confirmation dialog (shadcn AlertDialog): "Sign out?" / "You'll need to sign in again to access your workspace." / Cancel + Sign out buttons
3. On confirm: clear session, redirect to `/login`

### Theme toggle
1. Default: system preference
2. Toggle cycles: system -> light -> dark -> system
3. Persisted to localStorage
