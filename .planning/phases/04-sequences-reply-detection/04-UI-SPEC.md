## Screens

### Screen 1: Dashboard -- Replies Section (on `/` dashboard page)

**Position:** Between the intent feed section and the approval queue section, separated by 32px gaps. This is the "moment of delight" -- where users see repco working.

**Source:** CONTEXT.md decision: "Replies surface via Sonner toast + terminal header event + dedicated reply card in a new Replies section on the dashboard."

**Section Header:**
- "Replies" in Inter 28px semibold, with new reply count badge: shadcn `Badge` variant `secondary` showing number (e.g., "2")
- 8px gap between heading text and badge
- Margin-bottom: 16px

**Reply Card (stacked cards, consistent with signal-card and approval-card pattern):**
- Background: secondary surface
- Border: 1px border token, rounded-lg (8px radius)
- Padding: 16px
- Layout: vertical stack
- Gap between cards: 16px
- Sorted by reply detection time, newest first

Top row (horizontal flex, space-between):
- Left: platform badge (Reddit pill), prospect handle in Inter 20px semibold, 8px right of badge
- Right: "replied {time ago}" in Inter 14px regular, muted text

Original DM row (collapsed/truncated):
- Margin-top: 8px
- Label: "Your message:" in Inter 14px regular, muted text
- DM text: Inter 14px regular, muted text, max 1 line with text-overflow ellipsis
- Clicking expands to show full DM text (toggle)

Reply row:
- Margin-top: 12px
- Background: dominant surface (creates contrast from card secondary surface)
- Border-left: 3px solid accent color
- Padding: 12px 16px
- Reply text: Inter 16px regular, default text, line-height 1.5
- Full reply text displayed, no truncation

Action row:
- Margin-top: 16px
- Horizontal flex, 8px gap
- "View on Reddit" link: shadcn `Button` variant `outline`, size `sm`, opens in new tab
- "View prospect" link: shadcn `Button` variant `ghost`, size `sm`, navigates to prospect detail (Phase 5, disabled/hidden until then)

**Sequence stopped badge:**
- Below the reply row, 8px margin-top
- shadcn `Badge` variant `secondary` with green-500 text: "Sequence stopped -- reply received"

**Empty state (no replies yet):**
- Centered in replies section area
- Heading: "No replies yet" in Inter 16px weight 600
- Body: "When a prospect replies to your DM, it will appear here. repco checks inboxes every 2 hours." in Inter 14px regular, muted text

### Screen 2: Follow-up Cards in Approval Queue (on `/` dashboard page)

**Position:** Follow-up cards appear alongside initial DM approval cards in the existing approval queue section (Phase 3). They are visually identical except for a follow-up tag.

**Source:** CONTEXT.md decision: "Follow-up cards in approval queue have 24h expiry. Follow-up approval cards should be visually identical to initial DM approval cards -- same card layout, just with a Follow-up 2 of 3 label."

**Follow-up tag (addition to Phase 3 approval card):**
- Position: top-right of card, 16px from top edge, 16px from right edge
- shadcn `Badge` variant `outline` with accent border
- Text: "Follow-up {N} of 3" in Inter 14px medium
- N = 1 (day 3), 2 (day 7), 3 (day 14)

**Context addition (below post excerpt):**
- "Sent initial DM on {date}" in Inter 14px regular, muted text
- If previous follow-ups sent: "Follow-up 1 sent on {date}" in Inter 14px regular, muted text

All other card elements (post excerpt, intent score, angle, DM draft, action buttons, edit mode) are identical to Phase 3 approval card spec.

**Follow-up expiry:** 24h (vs 12h for initial DMs). Cards silently disappear when expired via Realtime update.

### Screen 3: Prospect Sequence Timeline (inline on prospect card/record)

**Source:** CONTEXT.md decision: "Prospect card shows mini-timeline: sent DM (done) -> follow-up 1 (day 3, pending/sent/skipped) -> follow-up 2 (day 7) -> follow-up 3 (day 14)."

**Position:** On the prospect record view (Phase 5 builds the full prospect detail page). For Phase 4, the timeline renders as a compact horizontal strip within the approval queue card context row when a follow-up is displayed, and on any future prospect detail view.

**Layout:** Horizontal flex, 8px gaps between steps

**Timeline step (4 steps total):**
- Dot: 12px circle, colored per Sequence Timeline Step Colors table
- Connecting line: 2px height, muted at 20% opacity, between dots
- Label below dot: "DM" / "Day 3" / "Day 7" / "Day 14" in Inter 14px regular
- State label (if not "pending"): "(sent)" / "(skipped)" / "(cancelled)" in Inter 14px regular, colored per step state

**"Stop sequence" button:**
- Position: right-aligned, inline with timeline
- shadcn `Button` variant `ghost`, size `sm`, destructive text color
- Text: "Stop sequence"
- Triggers confirmation dialog (AlertDialog)

**Stop sequence dialog:**
- Title: "Stop sequence for u/{prospect}?"
- Description: "This will cancel all remaining follow-ups. You can still send a new message manually."
- Cancel: "Keep sending" (default focus)
- Confirm: "Stop sequence" (destructive variant)

### Screen 4: Reply Detection Failure Warning Banner (on `/` dashboard page)

**Source:** CONTEXT.md decision: "Dashboard warning banner after any failed inbox check."

**Position:** Top of dashboard main content area, below terminal header, above all sections. Full width.

**Layout:**
- Background: amber-500 at 10% opacity
- Border: 1px solid amber-500 at 30% opacity
- Border-radius: rounded-md (6px)
- Padding: 12px 16px
- Horizontal flex, space-between

Left:
- Warning icon (phosphor `Warning`, 20px, amber-500)
- 8px gap
- Text: "Reply check failed for @{account} -- last successful check: {X}h ago" in Inter 14px regular, default text

Right:
- Dismiss button: phosphor `X` icon, 16px, muted text, ghost button styling
- On dismiss: banner hidden for this session only (reappears on refresh if still failing)

**Visibility rules:**
- Appears after any single failed inbox check
- Shows the most recent failure only (not stacked)
- Disappears automatically when the next inbox check succeeds

### Screen 5: Settings Page (`/settings`)

**Source:** CONTEXT.md decision: "Auto-send toggle lives in a global Settings page."

**Layout:** Sidebar (Phase 1 shell) + main content area. Vertical stack of settings sections.

**Page Header:**
- "Settings" in Inter 28px semibold
- Margin-bottom: 24px

**Section: Follow-up Sequences**
- Section heading: "Follow-up Sequences" in Inter 20px semibold
- Margin-bottom: 16px
- Background: secondary surface, rounded-lg, padding 24px

Auto-send toggle row:
- Horizontal flex, space-between, align-center
- Left side:
  - Label: "Auto-send follow-ups" in Inter 16px regular
  - Description: "When enabled, follow-up messages send automatically without your approval. Default cadence: day 3, 7, and 14." in Inter 14px regular, muted text
- Right side:
  - shadcn `Switch` component
  - Default: off (unchecked)

**Empty state (if no other settings exist yet):**
- Only the follow-up sequences section is shown
- No empty state needed -- the settings section is always visible with the toggle

## Interaction Contracts

### Replies section -- real-time updates
1. On dashboard load: fetch reply events (prospects with `pipeline_status = 'replied'`) sorted by reply detection time DESC
2. Subscribe to Supabase Realtime for prospect status changes to `replied`
3. New reply arrives: Sonner toast "u/{prospect} replied to your message", reply card prepends to Replies section with 300ms slide-down + fade-in
4. Agent card transitions to "Reply" emotional state
5. Terminal header receives "> Reply received from u/{prospect}" line
6. Reply count badge in section heading updates in real-time

### Follow-up cards in approval queue
1. Follow-up actions with `status = 'pending_approval'` appear in the existing approval queue (Phase 3 Realtime subscription handles this)
2. Follow-up cards are distinguished by the "Follow-up {N} of 3" badge and sequence context
3. Approve, Edit, Regenerate, Reject interactions are identical to Phase 3 approval card behavior
4. 24h expiry: follow-up cards silently disappear when expired via Realtime update (no toast)
5. When auto-send is enabled: follow-ups skip the approval queue entirely, terminal header shows "> Follow-up {N} sent to u/{prospect} via @{account}"

### Stop sequence
1. User clicks "Stop sequence" on prospect timeline
2. AlertDialog appears: "Stop sequence for u/{prospect}?"
3. On confirm: all pending follow-ups cancelled (status set to `cancelled`), timeline steps update to cancelled state
4. Toast: "Sequence stopped for u/{prospect}"
5. Cancelled timeline steps show strikethrough label with muted dot

### Reply stops sequence automatically
1. When reply detected: all pending follow-ups for that prospect cancelled immediately
2. Prospect `pipeline_status` updated to `replied`
3. Timeline steps update: completed steps stay green, all pending/future steps show cancelled state
4. "Sequence stopped -- reply received" badge appears on prospect record
5. Reply card appears in Replies section

### Reply detection failure warning
1. After any failed inbox check: warning banner appears at top of dashboard
2. Banner shows which account failed and how long since last success
3. User can dismiss banner (session only)
4. After 3 consecutive failures: email alert sent to user (NTFY-03)
5. Banner disappears automatically when next check succeeds

### Auto-send toggle
1. User navigates to `/settings` via sidebar
2. Toggle is off by default
3. User flips toggle: immediate save (server action, no save button)
4. Toast: "Auto-send enabled" or "Auto-send disabled"
5. When enabled: new follow-ups go directly to `approved` status, skip approval queue, execute via action engine
6. When disabled: new follow-ups go to `pending_approval` status, appear in approval queue

### Email notifications
1. Reply alert: sent within 10 minutes of reply detection (async, does not block dashboard update)
2. Daily digest: sent at 8:00 user's local timezone via scheduled cron
3. Account warning: sent immediately on status change to warning or banned
4. All emails sent via Resend with branded React email templates
5. No unsubscribe mechanism in V1 (per CONTEXT.md)

### Sidebar nav update
1. Add "Settings" nav item to sidebar below "Accounts"
2. Icon: phosphor `GearSix` at 16px
3. No notification badge on Settings nav item

### Terminal header -- Phase 4 events
1. Terminal header (from Phase 2/3) receives new event types
2. New line types: follow-up sent, follow-up scheduled, reply received, inbox check, inbox check failed
3. Same animation behavior as Phase 2/3 (fade-in at bottom, oldest scrolls out)
