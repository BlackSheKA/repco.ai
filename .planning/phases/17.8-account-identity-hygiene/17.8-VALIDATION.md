## Threat Model References

| Threat ID | Mitigation | Verification |
|---|---|---|
| T-17.5-02 (user input → Stagehand `act()` arg injection) | `resolveAccountIdentity` uses server-defined prompt + Zod schema; no user-controllable text reaches `extract()` or `act()` | Test: assert `extract()` is called with literal string prompt + literal Zod schema (no template-string interpolation); inspect call args in mocked Stagehand |
| T-17.8-01 (NEW — extracted identity collides with another user's account) | UNIQUE partial index `social_accounts_user_reddit_uidx` / `social_accounts_user_linkedin_uidx` (per-user); on `23505` collision, DELETE the new row + `deleteContext` BB context, surface "to konto jest już połączone" | Migration test inserts duplicate within same user → fails with 23505; integration test asserts row deletion + deleteContext call |
| T-17.8-02 (NEW — drift in Stagehand output bypasses regex) | Server re-validates extracted value against strict regex BEFORE writing; on failure, transition to `failed` (no DB write) | Unit test: feed `extract()` mock with malformed values (`"u/foo"`, `"x"` 1-char, `"name with space"`, empty) → assert no DB write + `identity_status='failed'` |
