## Component Inventory (per UI surface)

### Surface 1 — ConnectionFlow simplification

**File:** `src/features/accounts/components/connection-flow.tsx` and `account-list.tsx`.

**Removed:**
- Reddit handle input form (`account-list.tsx` lines 156–207, the entire `<form onSubmit={submitRedditHandle}>` block)
- `handleInput` state, `submitRedditHandle()`, `Label` + `Input` + `u/` prefix span
- The `connectAccount(platform, handle)` second arg — call site becomes `connectAccount(platform)`

**Behavior after change:**
- User on `/accounts` clicks `Connect Reddit Account` (or `Connect LinkedIn Account`) → `openConnectDialog(platform)` immediately calls `connectAccount(platform)` → on success, mounts `<ConnectionFlow>` with the Browserbase live view. Same flow LinkedIn already uses today.
- `ConnectionFlow` itself (`connection-flow.tsx`) is unchanged structurally. Step 1/2/3 layout, copy, iframe behavior, retry, cancel — all preserved.

**Loading states unchanged:** the existing `<Card>` with `Loader2` spinner + "Setting up your account..." copy now shows for both Reddit and LinkedIn (was LinkedIn-only).

### Surface 2 — AccountCard refit

**File:** `src/features/accounts/components/account-card.tsx` (left section, lines 167–177 — "Left section / display_name + verified-ago line").

**New rendering hierarchy in the left section:**

```
[MessageSquare icon] [display_name : text-xl font-semibold]  [PencilIcon button]  [HealthBadge]
                     [identity line: text-sm text-muted-foreground]
                     [created/verified timeline line: text-sm text-muted-foreground] (existing)
```

**Identity line (state machine):**

| identity_status | Rendered content |
|----|----|
| `pending` | `<Loader2 className="h-3 w-3 animate-spin" /> <span>Resolving identity…</span>` (both inside `text-sm text-muted-foreground`) |
| `resolved` (Reddit) | `<span>u/{reddit_username}</span> <ExternalLink className="h-3 w-3" />` — the whole line is an `<a>` to `https://reddit.com/user/{reddit_username}` with `target="_blank" rel="noopener noreferrer"`; `aria-label="Open u/{reddit_username} on Reddit"` |
| `resolved` (LinkedIn) | `<span>linkedin.com/in/{linkedin_slug}</span> <ExternalLink className="h-3 w-3" />` — anchor to `https://linkedin.com/in/{linkedin_slug}`; same target/rel; `aria-label="Open {linkedin_slug} on LinkedIn"` |
| `failed` | A shadcn `<Alert variant="destructive">` rendered BELOW the display_name+pencil row (full-width within the left section). Heading: `<AlertTitle>Identity check failed</AlertTitle>`. Body: `<AlertDescription>` copy + a `<Button size="sm" variant="default">` with `<RefreshCw className="mr-1 h-3.5 w-3.5" />Try again`. Disabled while in-flight; spinner replacement on click. The HealthBadge stays where it is — the failed banner is identity-specific, not a health override. |

**Why shadcn `Alert` for the failed state (not just text)** — this is a worker-blocking condition (account skipped by warmup/DM/intent until resolved). The Alert visually distinguishes a `failed` account from a `resolved` one at a glance, satisfying CONTEXT.md D-08: "should look visibly different from a `resolved` one so the user knows why their account isn't doing anything".

**Edit affordance (Pencil) placement and behavior:**
- Trigger: shadcn `<Button variant="ghost" size="icon" className="h-7 w-7">` containing `<Pencil className="h-3.5 w-3.5" />`. Placed inline immediately to the right of the display_name `<span>`, BEFORE the HealthBadge. `aria-label="Edit display name"`.
- Visible always (no hover-only reveal) — avoids discoverability traps. Subdued via `text-muted-foreground` so it doesn't compete with the display_name.

### Surface 3 — Edit interaction

**Choice: shadcn `Popover` anchored to the Pencil icon button. Modal/AlertDialog rejected.**

**Rationale:**
- Inline-editing-on-the-card without a popover would require swapping the `<span>` for an `<input>` in place, which fights with the card's flex layout and the verified/HealthBadge siblings on the same line. The card height would shift mid-row.
- A full `AlertDialog` is overweight for a 64-char free-text edit with no validation beyond non-empty + length cap. AlertDialogs in this codebase are reserved for destructive confirmations (Delete, Skip Warmup) — using one here trains users to expect "are you sure?" friction on a label edit, which is the wrong signal.
- Popover is contextually anchored (no spatial jump), single shadcn primitive, dismisses on outside-click or Escape (free a11y), and matches the lightweight "rename" affordance pattern users already understand from file managers / Notion.

**Inline-rename alternative — also acceptable** if the planner finds the popover anchoring fights the card's responsive layout collapse (`lg:flex-row`). In that case: swap the `<span>` for an `<Input>` at the same `text-xl font-semibold` size with `Save` / `Cancel` ghost buttons immediately to the right (replacing the Pencil position). Either is approved — popover is the default.

**Popover contents (when Pencil is clicked):**

```
┌─────────────────────────────────────┐
│ Display name                        │  <- Label, text-sm font-medium
│ ┌─────────────────────────────────┐ │
│ │ {current display_name}          │ │  <- shadcn Input, autoFocus, maxLength={64}
│ └─────────────────────────────────┘ │
│ {N}/64 characters                   │  <- text-xs text-muted-foreground
│                                     │
│         [ Cancel ]  [ Save ]        │  <- Save = variant="default", Cancel = variant="ghost"
└─────────────────────────────────────┘
```

- Width: `w-72` (288px). Padding: shadcn Popover default (`p-4`).
- Save is `disabled` when (a) trimmed value is empty, (b) trimmed value equals current display_name, or (c) request is in-flight. While in-flight, button content swaps to `<Loader2 className="mr-2 h-4 w-4 animate-spin" />Saving…`.
- Enter key in the Input submits (form-wrapped, with `e.preventDefault()`); Escape closes the popover.
- Server action: `updateDisplayName(accountId, value.trim())` returns `{ success, error }`. On `success`: close popover, fire toast (see Surface 4). On `error`: keep popover open, render `<p className="text-sm text-destructive">` below the input with the error string.
- Length cap is enforced both client-side (`maxLength={64}` on Input — hard browser cap) and server-side (server trims + rejects empty + length cap; UI never relies solely on client cap).

### Surface 4 — Toast contract (Sonner)

All toasts via the existing `toast` import from `sonner`. Match the existing tone (short, sentence-cased, no exclamation marks — see `feedback_landing_copy_style`).

| Trigger | Variant | Copy |
|---------|---------|------|
| `updateDisplayName` success | `toast.success` | `Display name updated` |
| `updateDisplayName` returns `{ error: "empty" }` | (rendered inside popover, NOT toast) | `Name can't be empty` |
| `updateDisplayName` returns `{ error: "too_long" }` | (rendered inside popover, NOT toast) | `Keep it under 64 characters` |
| `updateDisplayName` other error | `toast.error` | `Couldn't save the name — try again` |
| `retryAccountIdentity` triggered (button click, before result) | `toast.loading` (or none — instant button-spinner is enough) | `Resolving identity…` |
| `retryAccountIdentity` success | `toast.success` | `Identity resolved` |
| `retryAccountIdentity` failure (extract failed again) | `toast.error` | `Couldn't resolve identity — try again in a minute` |
| `retryAccountIdentity` returns `duplicate` (uniqueness collision per CONTEXT.md D-03) | `toast.error` | `This account is already connected` |

Inline (popover-local) errors are preferred over toasts for validation feedback — the user is already focused on the input. Toasts are reserved for state changes that affect the AccountCard outside the user's current focus.

## Accessibility Contract

- Pencil button: `aria-label="Edit display name for {display_name}"`.
- ExternalLink anchor (resolved state): wraps the entire identity line as one anchor; `aria-label="Open {identity} on {platform}"`; opens in new tab with `rel="noopener noreferrer"`.
- Pending spinner: container has `role="status" aria-live="polite"` so screen readers announce the resolving state once.
- Failed Alert: shadcn Alert already provides `role="alert"` — no extra wiring needed. Retry button has `aria-label="Retry identity resolution for {display_name}"`.
- Popover focus: Input `autoFocus`. On close (Save success or Cancel), focus returns to the Pencil trigger (shadcn Popover default behavior — verify in executor).
- Keyboard: Enter submits popover form; Escape cancels; Tab order is Input → Cancel → Save.
