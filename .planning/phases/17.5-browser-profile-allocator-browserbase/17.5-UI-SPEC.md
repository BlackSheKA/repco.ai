Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps inside flex rows (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `py-2`) |

**Iframe dimensions:** `width: 100%`, `height: 600px` (fixed). Rationale: 600px gives Reddit/LinkedIn login forms enough vertical room without forcing the parent dialog/card into viewport overflow on a 1024×768 minimum. Aspect-ratio fluid is rejected — login pages re-flow at narrow widths and break.

**Iframe container:** `rounded-md border bg-muted/30` — matches the existing `loginUrl` callout box treatment (line 159 in current file).

Exceptions: none.

## Interaction Contract — iframe live view

This is the load-bearing net-new pattern. Lock down exactly:

### Iframe element

```tsx
<iframe
  src={debuggerFullscreenUrl}
  sandbox="allow-same-origin allow-scripts allow-forms allow-popups"
  allow="clipboard-read; clipboard-write"
  className="w-full h-[600px] rounded-md border bg-muted/30"
  title={`${platformLabel} login session`}
/>
```

- **`sandbox`:** locked per CONTEXT.md "Claude's Discretion" → `allow-same-origin allow-scripts allow-forms allow-popups`. Required for: form submission (login), JS interaction (Browserbase debugger), and OAuth/2FA popup flows. Do NOT add `allow-top-navigation` — sandbox tightening deferred to security audit.
- **`title`:** screen-reader label, dynamically reflects platform. Example: "LinkedIn login session" or "Reddit login session".
- **`allow`:** clipboard read/write so users can paste passwords from a manager.
- **No `referrerPolicy` override** — Browserbase debugger URL handles its own auth via signed token.

### State machine (Step 1 only — Steps 2/3 unchanged)

| State | Trigger | Visual |
|-------|---------|--------|
| `starting` | Mount, before `startAccountBrowser` returns | Loader2 spinner + body text "Starting a remote browser for this account…" — unchanged from current |
| `error` | `startAccountBrowser` rejects or returns `success: false` | AlertTriangle (destructive) + heading "Could not start remote browser" + body shows `D-11` copy + Retry/Cancel buttons — unchanged from current |
| `ready` | `debuggerFullscreenUrl` received | Instructions list (revised, see below) + iframe + action row |
| `iframe_loading` | iframe mounted but `onLoad` not yet fired | Loader2 spinner overlaid on iframe container with `bg-muted/50 backdrop-blur-sm` — disappears on iframe load |

**`iframe_loading` overlay implementation:** absolutely-positioned div over the iframe container, removed via `onLoad` callback on the iframe element. Single `useState<boolean>` named `iframeLoaded`, default `false`, set `true` on iframe `onLoad`. Removed entirely on first load — no re-display on subsequent navigation inside the iframe.

### Instructions list (revised for embedded flow)

Replaces lines 147–194 (the current ordered list referencing "new tab" / "paste URL"):

```
1. Log into your {platformLabel} account in the browser below.
2. {platform === "linkedin" ? "If LinkedIn asks for 2FA or email verification, complete it here before clicking below." : null}
3. Once you see your home feed, click "I've logged in".
```

The platform-specific 2FA hint stays for LinkedIn, drops for Reddit (parity with current file).

The `loginUrl` clipboard-copy callout (current lines 156–177) is **deleted** — Browserbase navigates the session to the platform login URL server-side; the user does not paste anything.

### Action row

Three buttons, left-aligned, `flex flex-wrap gap-2`:

| Button | Variant | Label | Disabled when |
|--------|---------|-------|---------------|
| 1 | `default` (primary accent) | I've logged in | `iframeLoaded === false` |
| 2 | `ghost` | Cancel | never |

The "Open remote browser" external-link button is removed entirely. The "Retry" button stays only inside the `error` state branch.

## Copywriting Contract

All copy is sentence case (per memory `feedback_landing_copy_style`). User-facing error copy is **D-11 verbatim** — never expose vendor names, HTTP codes, or stack traces.

| Element | Copy |
|---------|------|
| Step indicator | `Step {step} of 3` — unchanged |
| Starting browser body | `Starting a remote browser for this account…` — unchanged |
| Error heading (start failure) | `Could not start remote browser` — unchanged structure |
| Error body (start failure) | `Could not set up the account right now — please try again in a moment.` — **D-11 locked**. Replace any vendor-specific error string from `startAccountBrowser` with this exact copy at render time. The server action result `error` field is NOT rendered. |
| Iframe title (a11y) | `${platformLabel} login session` |
| Iframe loading overlay | `Loading login screen…` |
| Instruction line 1 | `Log into your {platformLabel} account in the browser below.` |
| Instruction line 2 (LinkedIn only) | `If LinkedIn asks for 2FA or email verification, complete it here before clicking below.` |
| Instruction line 3 | `Once you see your home feed, click "I've logged in".` |
| Privacy reassurance | `repco never sees your password.` — render as muted footnote below the instructions |
| Primary CTA | `I've logged in` |
| Secondary action | `Cancel` |
| Verifying body (step 2) | `Verifying your session…` — unchanged |
| Verifying subtext (step 2) | `Checking {platformLabel} login status` — unchanged |
| Success heading (step 3) | `Account connected` — unchanged |
| Failure heading (step 3) | `Could not verify login` — unchanged |
| Failure body (step 3) | `Could not set up the account right now — please try again in a moment.` — **D-11 locked**. Override any error string. |
| Retry button (step 3 failure) | `Try logging in again` — unchanged |

**Empty state:** not applicable (this is a flow, not a list).

**Destructive confirmation:** none in this phase. `Cancel` calls `stopAccountBrowser(accountId)` server-side, which releases the Browserbase session and (per D-10 rollback) deletes the orphan context if no `social_account` row was committed. No confirm dialog — clicking Cancel is non-destructive from the user's mental model (they haven't "saved" anything yet).

## Accessibility

- iframe `title` attribute REQUIRED on every render (lints fail without it). Must reflect platform.
- Action-row primary button has `aria-disabled` matching `disabled` while `iframeLoaded === false`.
- Loading overlay uses `role="status"` + `aria-live="polite"` so screen readers announce "Loading login screen" once.
- Error states use existing icon + text pattern; no aria changes from current file.
- Focus management: when iframe finishes loading, do **not** auto-focus the iframe content (cross-origin, would fail silently and trap keyboard). Focus the "I've logged in" button instead via `useRef` + `requestAnimationFrame` after `iframeLoaded` flips true.
