## Screens

### Screen 1: Dashboard — Approval Queue Section (on `/` dashboard page)

**Position:** Below the intent feed section on the main dashboard, separated by 32px gap. Approval queue is a collapsible section with heading.

**Source:** CONTEXT.md decision: "Approval queue is a section/tab on the main dashboard page (not a separate route)."

**Section Header:**
- "Approval Queue" in Inter 28px semibold, with pending count badge: shadcn `Badge` variant `secondary` showing number (e.g., "3")
- 8px gap between heading text and badge
- Margin-bottom: 16px

**Approval Card (stacked cards, consistent with signal-card pattern):**
- Background: secondary surface
- Border: 1px border token, rounded-lg (8px radius)
- Padding: 16px
- Layout: vertical stack
- Gap between cards: 16px
- Sorted by recency, newest first

Top row (horizontal flex, space-between):
- Left: platform badge (same Reddit/LinkedIn pill from signal card), subreddit name, dot separator, author handle -- all using same styling as signal card
- Right: time ago in Inter 14px regular, muted text

Context row:
- Margin-top: 8px
- Original post excerpt: Inter 16px regular, default text, max 2 lines with text-overflow ellipsis
- Intent score: FlameIndicator component (reused from Phase 2), 8px gap after excerpt

Angle row:
- Margin-top: 8px
- "Suggested angle:" label in Inter 14px regular, muted text
- Angle text in Inter 14px regular, default text -- displayed inline after label

DM Draft row:
- Margin-top: 16px
- Background: dominant surface (creates contrast from card secondary surface)
- Border: 1px border token, rounded-md (6px radius)
- Padding: 16px
- DM text: Inter 16px regular, default text, line-height 1.5
- When editing: same area becomes a `textarea` with identical styling, auto-height, no resize handle

Action row:
- Margin-top: 16px
- Horizontal flex, 8px gap, right-aligned
- "Approve" button: shadcn `Button` variant `default` (accent background), size `sm`
- "Edit" button: shadcn `Button` variant `outline`, size `sm`
- "Regenerate" button: shadcn `Button` variant `outline`, size `sm`
- "Reject" button: shadcn `Button` variant `ghost`, size `sm`, destructive text color

**Edit mode (inline):**
- Triggered by clicking "Edit" button
- DM text area becomes editable textarea
- "Edit" button label changes to "Discard edits"
- "Approve" button remains visible (approves edited text)
- Textarea auto-focuses on edit activation
- No modal, no separate screen

**Empty state (no pending approvals):**
- Centered in approval queue section area
- Heading: "No messages pending" in Inter 16px weight 600
- Body: "When you click Contact on a signal, repco will draft a DM for your review." in Inter 14px regular, muted text

### Screen 2: Accounts Page (`/accounts`)

**Source:** CONTEXT.md decision: "Dedicated /accounts route in sidebar nav (separate from main dashboard)."

**Layout:** Sidebar (Phase 1 shell) + main content area. Vertical stack of account cards.

**Page Header:**
- "Accounts" in Inter 28px semibold
- Margin-bottom: 24px

**Account Card (one per connected social account):**
- Background: secondary surface
- Border: 1px border token, rounded-lg (8px radius)
- Padding: 24px
- Gap between cards: 16px
- Layout: responsive -- horizontal on desktop, vertical stack on mobile

Left section:
- Reddit icon (phosphor `RedditLogo`) at 32px, muted text color
- Username: Inter 20px semibold, 8px right of icon, vertically centered
- Health badge: shadcn `Badge` with custom colors per Health Status Badge Colors table, 8px right of username
- Last action: "Last action: {time ago}" in Inter 14px regular, muted text, below username row with 4px top margin

Center section -- Warmup Progress:
- Label: "Warmup: Day {X} of 7" in Inter 14px regular
- Progress bar: full width of center section, 8px height, rounded-full
- Progress bar background: muted at 20% opacity
- Progress bar fill: colored per Warmup Progress Bar Colors table, segmented
- If warmup complete: "Warmup complete" in Inter 14px regular, green-500 text, no progress bar
- If warmup skipped: "Warmup skipped" in Inter 14px regular, amber-500 text with tooltip "This account skipped warmup. Higher ban risk."

Right section -- Daily Limits:
- Three columns, 16px gap
- Each column: number over label layout
  - Number: Inter 20px regular (e.g., "3/8")
  - Label: Inter 14px regular, muted text (e.g., "DMs")
- Columns: "DMs" (used/8), "Engage" (used/20), "Replies" (used/5)
- When at limit: number text turns amber-500
- When over 80% of limit: number text turns amber-500 at 60% opacity as warning

**Empty state (no connected accounts):**
- Centered on page
- Heading: "No accounts connected" in Inter 20px semibold
- Body: "Connect a Reddit account to start sending messages. Each account gets a unique browser profile for safety." in Inter 16px regular, muted text
- CTA: "Connect Reddit Account" -- shadcn `Button` variant `default`, size `default`

**Warmup Skip Dialog:**
- Triggered by a "Skip warmup" text button (ghost variant, 14px) below the warmup progress bar
- Uses shadcn `AlertDialog`
- Title: "Skip warmup for u/{username}?"
- Description: "Skipping warmup increases the risk of your account being flagged or banned by Reddit. Only do this if this account already has recent activity."
- Cancel: "Keep warming up" (default focus)
- Confirm: "Skip warmup" (destructive variant)

### Screen 3: Account Connection Verification (within `/accounts`)

**Position:** Inline within the accounts page, appears when user initiates account connection.

**Connection flow card:**
- Same card styling as account cards
- Step indicator: "Step {N} of 3" in Inter 14px regular, muted text

Step 1 -- GoLogin Login:
- Instruction: "Log into Reddit using the GoLogin browser window that just opened." in Inter 16px regular
- Sub-instruction: "Use your Reddit credentials. repco never sees your password." in Inter 14px regular, muted text
- "I've logged in" button: shadcn `Button` variant `default`, size `default`

Step 2 -- Verification:
- Instruction: "Verifying your session..." in Inter 16px regular
- Spinner icon + "Checking Reddit login status" in Inter 14px regular, muted text
- Auto-advances on success

Step 3 -- Result:
- On success: green checkmark icon (phosphor `CheckCircle`, 32px, green-500) + "Account connected" in Inter 16px semibold
- On failure: amber warning icon (phosphor `WarningCircle`, 32px, amber-500) + "Could not verify login" in Inter 16px semibold + "Try logging in again" button (outline variant)

## Interaction Contracts

### Approval queue -- real-time updates
1. On dashboard load: fetch pending approval actions (`status = 'pending_approval'`) sorted by `created_at DESC`
2. Subscribe to Supabase Realtime `actions` table for inserts and updates filtered by user's workspace
3. New approval arrives: card prepends to queue with 300ms slide-down + fade-in (same animation as signal cards)
4. Action approved/rejected: card fades out over 200ms and collapses (height to 0 over 200ms)
5. Pending count badge updates in real-time

### Approve action
1. User clicks "Approve" on approval card
2. Button enters loading state (spinner, disabled), all other action buttons disabled
3. Action status updated to `approved` in database
4. Card fades out and collapses
5. Toast: "Message approved -- sending shortly"
6. Terminal header receives execution event when action begins executing

### Edit action (inline)
1. User clicks "Edit" on approval card
2. DM draft area becomes editable textarea, auto-focused, cursor at end
3. "Edit" button label changes to "Discard edits"
4. "Approve" button remains active -- approves the edited text
5. User clicks "Discard edits" -- textarea reverts to original text, returns to read-only
6. User clicks "Approve" -- saves edited text, then follows approve flow

### Regenerate action
1. User clicks "Regenerate" on approval card
2. Button enters loading state (spinner), DM draft area shows skeleton pulse
3. Backend generates new DM via Claude Sonnet 4.6 with different angle
4. New draft replaces old text with 200ms fade transition
5. Toast: "Generating a new draft..."

### Reject action
1. User clicks "Reject" on approval card
2. No confirmation dialog (low-stakes action, prospect stays in pipeline)
3. Action status updated to `rejected`
4. Card fades out and collapses
5. Toast: "Message rejected"

### Expired actions
1. Actions older than 12 hours with status `pending_approval` are set to `expired` by backend
2. Expired cards silently disappear from queue via Realtime update (no countdown shown to user)
3. No toast or notification for expiry

### Account health -- real-time updates
1. On `/accounts` load: fetch all social accounts for user
2. Subscribe to Supabase Realtime `social_accounts` table for updates
3. Health status change: badge updates in-place with 200ms color transition
4. Sonner toast: "{username} status changed to {status}"
5. Red badge dot appears on "Accounts" sidebar nav item when any account is warning or banned

### Warmup progress
1. Progress bar fills proportionally based on current warmup day (day/7 * 100%)
2. Updates once daily (no real-time animation needed)
3. On warmup completion (day 8): progress bar replaced with "Warmup complete" label + green checkmark

### Account connection
1. User clicks "Connect Reddit Account" on accounts page
2. Connection flow card appears at top of account list
3. Backend creates GoLogin Cloud profile via API
4. GoLogin web dashboard opens in new tab for user to log in
5. User clicks "I've logged in" -- step 2 begins
6. Backend opens GoLogin profile headlessly, navigates to Reddit, checks session
7. On success: step 3 shows green checkmark, card transitions to normal account card after 2s
8. On failure: step 3 shows retry prompt, user can try again

### Sidebar nav update
1. Add "Accounts" nav item to sidebar below "Approvals" (or below existing nav items)
2. Icon: phosphor `UserCircle` at 16px
3. Red notification dot on nav item when any account has `warning` or `banned` status
4. Notification dot: 8px circle, `#DC2626`, positioned top-right of icon

### Terminal header -- Phase 3 events
1. Terminal header (from Phase 2) now receives real action execution events
2. New line types: DM sent, action executing, action failed
3. Same animation behavior as Phase 2 (fade-in at bottom, oldest scrolls out)

| Element | Requirement |
|---------|-------------|
