  <action>
Edit `src/app/(app)/layout.tsx` to query social_accounts for the authenticated user and determine whether any account has non-healthy status. Pass the boolean to AppShell.

After the existing `supabase.auth.getUser()` block and the `if (!user) redirect("/login")` guard, add a query before the `return`:

```typescript
const { count: alertCount } = await supabase
  .from("social_accounts")
  .select("id", { count: "exact", head: true })
  .eq("user_id", user.id)
  .in("health_status", ["warning", "cooldown", "banned"])

const hasAccountAlerts = (alertCount ?? 0) > 0
```

Then update the AppShell JSX:
```typescript
return (
  <AppShell
    user={{ email: user.email ?? "" }}
    terminalHeader={<TerminalHeader userId={user.id} />}
    hasAccountAlerts={hasAccountAlerts}
  >
    {children}
  </AppShell>
)
```

Do NOT change imports other than what's needed (none should be needed — supabase and AppShell are already imported). Do NOT remove the auth redirect. Do NOT refactor unrelated code.

Notes:
- Use `{ count: "exact", head: true }` — we don't need rows, only the count. This is the cheapest possible query.
- RLS policies on social_accounts already restrict to the user's own rows (per Phase 1) — the `.eq("user_id", user.id)` is redundant but defensive.
- If the query errors (e.g., RLS edge case), `alertCount` will be null → coalesce to 0 → hasAccountAlerts=false (fail-safe: no false red dot).
