<objective>
Fix `worker.ts` so every action execution writes exactly one schema-valid `job_logs` row, covering all 5 early-failure paths plus the normal CU success/fail paths.

Purpose: The current `job_logs` insert at line 293–299 references non-existent columns `details` and `correlation_id`. PostgREST silently drops unknown columns, so action rows never reach `job_logs`. OBSV-04 threshold alerts read `job_logs` for success/fail rate — the fix restores those alerts' input.

Output: `worker.ts` with a `try/finally` structure. Shared local state (`runStatus`, `runError`, `cuSteps`, `screenshotCount`, `platform`, `actionType`, `userId`) is accumulated along the pipeline path taken. The `finally` block performs exactly one `job_logs` insert with schema-valid columns. Early `return` paths inside `try` are refactored to set shared state instead of returning directly, then break/fall-through to `finally`. The outside-active-hours re-queue path remains a plain `return` before `try` (no `job_logs` row written).
</objective>

job_logs schema (from supabase/migrations/00002_initial_schema.sql §11):
```sql
CREATE TABLE job_logs (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  job_type     job_type NOT NULL,           -- ENUM: 'monitor' | 'action' | 'reply_check'
  status       job_status_type NOT NULL,    -- ENUM: 'started' | 'completed' | 'failed' | 'timeout'
  user_id      uuid REFERENCES users ON DELETE SET NULL,
  action_id    uuid REFERENCES actions ON DELETE SET NULL,
  started_at   timestamptz DEFAULT now(),
  finished_at  timestamptz DEFAULT now(),
  duration_ms  integer,
  error        text,
  metadata     jsonb
);
```

Valid status values for worker: `"completed"` (CU success), `"failed"` (CU failure OR any early failure).
NOT valid (do not use): `"details"`, `"correlation_id"` — these do not exist as top-level columns.

Canonical pattern from src/app/api/cron/zombie-recovery/route.ts:
```typescript

  started_at: startedAt.toISOString(),
  finished_at: new Date().toISOString(),
  duration_ms: durationMs,

})
```
